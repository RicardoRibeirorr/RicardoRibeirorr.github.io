Being a great software developer is not an easy job, as we all know. You must possess a wealth of knowledge and work hard to advance. However, during my work, I discovered what distinguishes good and outstanding software engineers, and here are some tips on how to become one. If you're interested in this topic then continue reading.

Master one programming language in depth

Give one programming language a thorough examination. Be a master at it by learning everything you can about it. Today, you can choose among good programming languages like Python, Java, C#, and Rust.

Learn Software Engineering concepts

Following your mastery of a programming language's syntax, semantics, and structures, you should also understand other software engineering ideas, including:

  • Software architecture
  • Software design
  • Design patterns
  • Clean code
  • SOLID, DRY, KISS, and YAGNI principles

This stands out in an interview, and shows that you are a great software engineer.

Learn the complete SDLC process

The entire software development process, from requirements through deployment, should be understood. Learn about DevOps, Quality assurance, and Agile techniques. Additionally, try working on a variety of projects; the more projects you work on, the more you'll learn and develop.

Be a product-minded engineer

Don't merely stick to the specifications and start implementing it when working on your project. Consider other suggestions, then provide them to your product manager. Try to comprehend the entire system as well as how business is conducted. Own the whole product feature lifecycle.

Improve your communication and soft-skills

Be humble, communicate clearly, and show others respect. Although showing kindness has no monetary cost, the benefits are immeasurable. Program in a group or in pairs. Grab a coffee or lunch with some people who aren't in engineering, or just talk to them in the hallway.

Learn intentionally

The tricky part is knowing when and how to learn. Don't just absorb information because that is ineffective. To have the biggest impact, we must intentionally learn things right before we need them.

Work with someone more experienced

Finding a mentor is the best approach to advance in your profession quickly. He can show you some fresh opportunities and assist you in identifying your deficiencies. There are mentors both inside and outside of organizations (check specialized services).

Use good engineering practices

Learn and follow good practices', such as:

  • Using version control
  • Write your tests properly (check the test pyramid)
  • Learn how to refactor
  • Learn TDD
  • Code reviews
  • DevOps mindset

Also, learn your IDE very well, use IDE extensions, and all the important shortcuts you need. Debugger too.

Use productivity techniques

To be more productive, we need to learn different techniques, such as:

  • Prioritization (check Eisenhower matrix)
  • Time management (check Pomodoro)
  • Concentration (check Deep focus / no distractions)
  • Note taking (check Notion)
