OWAP-Top-10-For-Beginners

1. Injection

Injection flaws occur when untrusted data is sent to an interpreter as part of a command or query. This can lead to the execution of unintended commands or unauthorized access to data. Common types include SQL injection, NoSQL injection, and LDAP injection.

Example of SQL Injection:

            
String query = "SELECT * FROM users WHERE username = '" + username + "' AND password = '" + password + "'";
ResultSet result = statement.executeQuery(query);

2. Broken Authentication

Broken authentication vulnerabilities occur when attackers compromise user credentials or session tokens, enabling unauthorized access to user accounts. This can lead to account takeover, unauthorized access to sensitive information, and more.

Example of Multi-factor Authentication (MFA):

            
public void authenticateUser(String username, String password, String otpCode) {
    // Verify username and password
    // Verify OTP code
    // Allow access if all verifications pass
}

3. Sensitive Data Exposure

Sensitive data exposure occurs when sensitive information is not adequately protected. This can include passwords, credit card numbers, health records, and more. When exposed, this data can be exploited by attackers for identity theft, financial fraud, and other malicious activities.

Example of Encryption of Data-at-Rest:

            
AES256.encrypt(data, encryptionKey);

4. XML External Entities (XXE)

XXE vulnerabilities occur when an application parses XML input, allowing attackers to access files on the server or interact with internal systems. This can lead to unauthorized data disclosure, server-side request forgery (SSRF), and more.

Example of Disabling External Entity Processing:

            
DocumentBuilderFactory factory = DocumentBuilderFactory.newInstance();
factory.setFeature(XMLConstants.FEATURE_SECURE_PROCESSING, true);
